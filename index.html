Javascript is an interpreted, single-threaded, single concurrent, and lightweight programming language, similar to other scripting languages. In other words, it has a single Call Stack.Therefore it can do only one thing at a time. Javascript uses dynamic typing and type safety verified at the runtime means we do not have to compile the JavaScript source code before sending it to the browser. An interpreter takes the raw JavaScript code and runs it for us. As we know, asynchronous programming is excellent for the faster execution of programs, and JavaScript is asynchronous by nature and designed to ensure non-blocking code execution. As we know, non-blocking code does not prevent another piece of code execution.

in a nutshell, JavaScript is a full-fledged programming language that can do most things a regular language like Java Python can do, such as
  • Declaring variables.
  • Storing and retrieving values.
  • Defining and invoking functions, including arrow functions.
  • Defining JavaScript objects and classes.
  • Loading and using external modules.
  • Writing event handlers that respond to click events.
  • Writing server code.
  • And much more.

localStorage and sessionStorage are almost identical and have the same API. One significant difference between localStorage and sessionStorage is the expiration date.
  • They have a limit of around 5MB of storage space.
  • They store key/value pairs for a particular domain.
  • The values stored may only be strings.
Difference
localStorage
  • Stores data with no expiration date, which means until we set an expiration date or clean the data by opening the developer console, it will be there for current and future visits to the site
    • sessionStorage
  • Stores data for one session, which means once the browser or the tab is closed, session storage will be no longer available

    • The event loop concept is straightforward. JavaScript Event Loop monitors the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and push it to the Call Stack, which effectively runs. Otherwise, If the callback queue is empty, nothing will happen. In simple words, an event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty. As we know, JS executed all its operations on a single thread but using intelligent data structures. The event loop gives us the flavor of multi-threading.
